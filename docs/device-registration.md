# Device Registration Module

Desktop agents register through `POST /api/v1/devices` using a user JWT. A
registered device receives a one-time pairing code that must be confirmed before
Socket.IO agent connections are accepted.

## Device Fields

- `name`, `hostname`, `platform`, `device_fingerprint`
- `agent_version`, `os_version`, `capabilities`
- `status`, `last_heartbeat_at`, `is_paired`

## Endpoints

- `GET /api/v1/devices`
- `POST /api/v1/devices`
- `GET /api/v1/devices/<device_id>`
- `PATCH /api/v1/devices/<device_id>`
- `DELETE /api/v1/devices/<device_id>`
- `POST /api/v1/devices/<device_id>/heartbeat`
- `POST /api/v1/devices/<device_id>/pair`
- `POST /api/v1/devices/<device_id>/pairing-code`

## Safety

The backend refuses Socket.IO connections for unpaired devices. Pairing codes are
stored only as bcrypt hashes and can be regenerated by the account owner.
